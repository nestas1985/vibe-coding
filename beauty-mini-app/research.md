## 3. Лучший UX — обязательные экраны

### Для клиента
1. **Профиль мастера** — фото, услуги с ценами, портфолио, рейтинг, кнопка «Записаться»
2. **Выбор услуги** — список с ценой и длительностью
3. **Выбор даты + времени** — в одном экране, только свободные слоты в реальном времени
4. **Подтверждение** — итог + предоплата
5. **Мои записи** — ближайшая запись, история, кнопка отмены
6. **Напоминания** — за 24 часа и за 2 часа до визита

### Для мастера
1. **Дашборд** — записи сегодня, выручка дня, быстрые действия
2. **Расписание** — недельный/дневной вид, цветовые метки
3. **Добавить запись** — клиент из базы или новый
4. **База клиентов** — история, заметки, аллергии, предпочтения
5. **Услуги и цены** — каталог, длительность каждой услуги
6. **Портфолио** — загрузка фото работ по категориям
7. **Настройки** — рабочие часы, перерывы, выходные

### Золотое правило записи
```
Экран 1: Профиль мастера → выбираешь услугу
Экран 2: Выбор даты + времени (в одном экране)
Экран 3: Подтверждение + оплата
```
**Цель: меньше 30 секунд и 3 шага. Без обязательной регистрации до записи.**

### Система напоминаний (снижает no-show на 30–40%)
| Момент | Канал | Что отправлять |
|--------|-------|----------------|
| После записи | Push / Telegram | Подтверждение: мастер, время, адрес |
| За 24 часа | Push + SMS | Напоминание + «Отменить без штрафа» |
| За 2 часа | Push | Короткий напоминальщик |
| После визита | Push | «Оцените работу» |

- Нюдовые тона + золото, или глубокий фиолетовый + розовый акцент
- Градиенты и glassmorphism (полупрозрачные карточки)

- Крупные display-заголовки для первого экрана
- Рекомендуемый стек: **Playfair Display + Inter** или **Cormorant Garamond + Montserrat**
- Letter-spacing в заголовках — ощущение люкса

### UI-паттерны
- Большие фото мастеров и работ
- Нижняя навигация (tab bar) с крупными иконками
- Мягкие скругления карточек (border-radius 16–24px)
- Instagram-style сетка для портфолио (3 колонки)
- Before/After слайдер для демонстрации работ
- Тэги услуги прямо на фото: «Гель-лак, 1500 руб.»

### Что отличает хорошее от плохого
| Хорошее | Плохое |

| Запись за 30 секунд | Обязательная регистрация сразу |

## 7. Экспертная оценка

### 7.1 UX-дизайнер

#### Ошибки новичков — то, что сразу убивает приложение

- **Обязательная регистрация перед записью.** Клиент зашёл посмотреть свободное время — ему предложили ввести имя, телефон, email. Он закрыл. Потеряли 60–70% пользователей на этом шаге.
- **Все 40 услуг в одном списке без категорий.** Маникюр, педикюр, наращивание, снятие, коррекция — вперемешку. Клиент не понимает, что выбрать.
- **Кнопка «Записаться» внизу длинного скролла.** Пока человек листает прейскурант, желание записаться остывает. Кнопка должна быть видна всегда.
- **Нет реальной доступности.** Показывают сетку на неделю, а половина слотов занята — клиент узнаёт об этом только после нажатия. Раздражает.
- **5 шагов вместо 3.** Каждый лишний экран — это потеря части людей. Выбор мастера и даты должны быть на одном экране.
- **Нет подтверждения после записи.** Клиент нажал «Готово» — тишина. Он не понимает, записался ли вообще.

#### Что обязательно должно быть

- Фото работ мастера **на первом экране** — без этого незнакомый мастер не вызывает доверия
- Реальные свободные слоты, без «уточняйте у мастера»
- Запись в 3 шага: услуга → время → подтверждение
- Мгновенное сообщение в Telegram после записи с деталями
- Кнопка отмены доступна клиенту самостоятельно — без звонка

#### Что бесит пользователей — конкретные примеры из отзывов

- **Слишком много напоминаний.** Fresha получает жалобы: «4 уведомления за 30 минут до визита». Оптимум: за 24 часа + за 2 часа, не больше.
- **Нет возможности выбрать конкретного мастера** — только «первый свободный». Клиентки beauty-мастеров привязаны к человеку, не к услуге.
- **Мелкие кнопки.** На мобильном слот 9:00 рядом с 9:30 — попасть пальцем сложно. Минимум высота кнопки 48px.
- **Принудительный редирект на скачивание приложения** при попытке записаться через браузер. Telegram Mini App решает эту проблему — всё внутри.
- **Нет возможности написать мастеру.** Клиент хочет уточнить — кнопки нет, только звонок. Добавить ссылку на чат с мастером обязательно.

### 7.2 Маркетолог

#### Что заставит клиента записаться

- **Портфолио работает лучше любого текста.** Фото «до/после» ногтей убеждает быстрее, чем описание услуги. Клиент должен увидеть работу мастера ещё до первого нажатия.
- **Социальное доказательство.** «Записались на этой неделе: 23 человека» или «47 постоянных клиентов» — снижает тревогу у новых.
- **Эффект дефицита.** «Осталось 2 свободных места на следующей неделе» — работает, если это правда. Если врать — доверие рушится навсегда.
- **Имя мастера везде, не «Салон».** Мастера красоты — личный бренд. Клиент записывается к Алёне, а не в «Nail Studio».
- **Первая запись — специальная цена.** Скидка 10–15% для нового клиента снимает барьер попробовать.

#### Элементы, которые повышают конверсию

| Элемент | Почему работает |
|---------|----------------|
| Цена видна сразу, без «уточняйте» | Убирает неопределённость — главный враг конверсии |
| Реальные фото работ (не стоковые) | Доверие и желание записаться именно к этому мастеру |
| Рейтинг + число отзывов | «4.9 — 127 отзывов» убеждает лучше любого текста |
| Кнопка «Записаться» крупная и заметная | Клиент не должен искать, как записаться |
| Автозаполнение для постоянного клиента | Убирает трение при повторной записи |
| Напоминание «Вы давно не приходили» | Возвращает клиентов без рекламы |
| Ссылка на профиль мастера из Instagram | Трафик из соцсетей сразу в запись |

#### Главная ошибка маркетинга в beauty-приложениях

Делают красивый дизайн, но прячут самое главное — **цену и доступность**. Клиент листает три экрана, прежде чем узнать, сколько стоит маникюр и есть ли запись в субботу. К этому моменту половина уже ушла. Цена и ближайший свободный слот должны быть на первом экране.

#### Рычаги удержания (retention)

- День рождения клиента → автосообщение со скидкой
- «Прошло 3 недели с вашего последнего визита» → напоминание
- Повторная запись в 1 клик прямо в сообщении после визита
- Программа: «5-й визит — скидка 20%» — видна в профиле клиента

### 7.3 Telegram-разработчик

#### Ограничения, которые нужно знать с самого начала

- **Mini App не может сам отправить уведомление.** Только через бота. Бот — обязателен, Mini App — интерфейс поверх него. Это разные вещи.
- **Загрузка файлов через Telegram API — максимум 20 МБ.** Фото портфолио надо хранить на внешнем сервере (S3, Cloudflare R2), не в Telegram.
- **CloudStorage (хранилище данных Mini App) — лимит 1024 записи.** Хранить там базу клиентов нельзя. Нужен бэкенд и база данных.
- **WebView в Telegram тормозит на слабых телефонах.** Тяжёлые анимации и CSS-эффекты — с осторожностью.
- **Нельзя открыть внешний URL напрямую** без подтверждения пользователем (Telegram показывает предупреждение). Это мешает редиректу на оплату — надо использовать нативный Telegram Payments.

#### Что работает хорошо — использовать обязательно

- **`MainButton`** — нативная кнопка Telegram внизу экрана. Выглядит как часть интерфейса, не нужно верстать. Для кнопки «Записаться» — идеально.
- **`BackButton`** — работает с жестом «назад» на Android. Пользователь ожидает именно такое поведение.
- **`HapticFeedback`** — лёгкая вибрация при нажатии. Ощущение нативного приложения, а не сайта.
- **`initData`** — при открытии Mini App Telegram передаёт данные пользователя (имя, username, id). Авторизация автоматически — не нужен отдельный логин.
- **`themeParams`** — приложение автоматически наследует тему Telegram (тёмная/светлая). Не нужно делать два дизайна.
- **Telegram Payments** — оплата не выходя из Telegram, через ЮKassa / Stripe. Конверсия выше, чем редирект на сторонний сайт.

#### Что лучше не делать

- **Не делать горизонтальную прокрутку.** В WebView она глючит и конфликтует со свайпами Telegram.
- **Не подключать внешние шрифты (Google Fonts) без fallback.** При медленном интернете страница загрузится без шрифта и будет выглядеть сломанной.
- **Не делать автозапуск видео или звука** — Telegram блокирует autoplay.
- **Не верстать под десктоп.** 95%+ пользователей Telegram в России — мобильные. Десктопный вид — бонус, не основа.
- **Не делать сложные CSS-анимации с `backdrop-filter`** (glassmorphism) везде — тормозит на слабых устройствах. Использовать точечно.
- **Не хранить чувствительные данные в `localStorage`** — в WebView Telegram это не безопасно. Всё важное — на бэкенде.
- **Не делать бот и Mini App независимыми.** Они должны работать как единая система: бот отправляет напоминания, Mini App — интерфейс для управления.

#### Технический стек для старта

```
Фронтенд:    Vanilla JS + HTML/CSS (без фреймворков — быстрее загружается)
             или React если много интерактивных состояний
Бэкенд:      Node.js (aiogram если Python) + база данных PostgreSQL
Хранение:    Cloudflare R2 или Yandex Object Storage для фото
Оплата:      ЮKassa через Telegram Payments API
Деплой:      Railway / Render (бэкенд) + Cloudflare Pages (фронтенд)
```
