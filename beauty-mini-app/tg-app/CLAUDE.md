# Beauty Mini App — Документация для разработчика

## Структура файлов

```
tg-app/
├── index.html          — точка входа, подключает SDK и скрипты
├── css/
│   └── styles.css      — все стили (переменные, компоненты, экраны)
└── js/
    ├── data.js         — данные: мастер, услуги, портфолио, клиенты
    └── app.js          — вся логика: навигация, рендер, события
```

---

## Как работает навигация

Приложение — Single Page App. Все экраны рендерятся в `<div id="screen">`.

**Переход вперёд:**
```javascript
goTo('services')           // сохраняет текущий экран в S.history
```

**Назад:**
```javascript
navigateBack()             // достаёт предыдущий экран из S.history
```

**Анимация:** CSS-классы `slide-in-right` / `slide-in-left` — 260ms.

## Где менять данные

**Профиль мастера** → `js/data.js`, объект `MASTER`

**Услуги** → `js/data.js`, массив `SERVICES`
- `id` — уникальный номер
- `category` — группировка на экране
- `price` — цена в рублях
- `duration` — время в минутах
- `emoji` — иконка

**Портфолио** → `js/data.js`, массив `PORTFOLIO`
- Сейчас: градиентные карточки. Чтобы поставить фото — замени `gradient` на `url('...')`

**Демо-клиенты** → `js/data.js`, массив `DEMO_CLIENTS`

## Как работают события

Один делегированный слушатель на `#screen`. Каждый интерактивный элемент имеет:
```html
<button data-a="действие" data-v="значение">
```

`data-a` — название действия (action), `data-v` — параметр.

Все действия → функция `dispatch()` в `app.js`.

## Хранилище

Записи сохраняются в `localStorage` под ключом `beauty_bookings`.

Заметки о клиентах: `notes_{id}`.

При первом запуске загружаются демо-записи из `getDemoBookings()` в `data.js`.

## Телеграм-интеграция

- `tg.ready()` + `tg.expand()` — при старте
- `tg.BackButton` — нативная кнопка «Назад» Telegram
- `tg.HapticFeedback` — вибрация на каждое нажатие
- `tg.themeParams` — цвета темы (тёмная/светлая автоматически)
- `tg.initDataUnsafe.user` — имя пользователя подставляется в форму

**Что добавить в v2:**
- `tg.MainButton` — нативная кнопка подтверждения внизу
- `Telegram Payments` — оплата через ЮKassa
- Уведомления через бота (Node.js бэкенд)

## Режимы приложения

- **Клиент** — главный экран `home`, флаг `S.isMaster = false`
- **Мастер** — главный экран `master-home`, флаг `S.isMaster = true`

Переключение: кнопка «💼 Мастер» / «👤 Выход» в хедере (для демо).

В продакшне: определять по `tg.initDataUnsafe.user.id` — если совпадает с id мастера, показывать мастер-режим.
