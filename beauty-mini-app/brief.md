# Beauty Mini App — Бриф разработки

> Основан на: research.md | Версия: MVP 1.0 | Язык: русский

---

## Концепция одной фразой

Telegram Mini App для частного мастера красоты: клиент записывается в 3 нажатия, мастер видит расписание и не отвечает на вопросы вручную.

---

## Требования к дизайну (не нарушать)

- Стиль — нативный Telegram, не сайт. Использовать `themeParams` чтобы цвета подстраивались под тему пользователя
- Кнопки — минимум 44px высотой, лучше 56px. На каждом экране максимум 1 главная кнопка
- Текст — минимум. Фото, иконки, цвет — максимум
- Шрифт системный (SF Pro на iOS, Roboto на Android) — как в Telegram
- Скругления карточек — 16px, тени лёгкие
- Нижняя навигация через `MainButton` Telegram там где нужно подтверждение
- `BackButton` Telegram — вместо своей стрелки "назад"
- `HapticFeedback` на каждом нажатии кнопки
- Никаких горизонтальных скроллов
- Никаких внешних шрифтов (Google Fonts и т.п.)

---

## Два режима приложения

Одно приложение — два входа:

- **Клиент** — открывает ссылку или бота, видит профиль мастера и записывается
- **Мастер** — открывает то же приложение, но с правами владельца. Управляет расписанием и видит список записей

Как различить: при первом входе проверяем `initData.user.id` — если это id мастера, показываем интерфейс мастера.

### Экран 01 — Профиль мастера
*Это первое, что видит клиент. Должно захотеться записаться.*

**Что на экране:**
```
┌─────────────────────────────┐
│  [Фото мастера — круглое,   │
│   большое, по центру]       │
│                             │
│  Имя Фамилия                │
│  Мастер ногтевого сервиса   │
│  ★ 4.9  · 127 отзывов       │
│                             │
│  ─── Портфолио ───          │
│  [фото][фото][фото]         │
│  [фото][фото][фото]  →      │
│                             │
│  ─── Услуги ───             │
│  Маникюр классический 1500₽ │
│  Маникюр гель-лак    2200₽  │
│  Педикюр             2500₽  │
│                             │
│  [ЗАПИСАТЬСЯ]  ← MainButton │
└─────────────────────────────┘
```

**Элементы:**
- Фото мастера — круглое, диаметр 96px, по центру
- Имя (крупно, 22px) + специализация (14px, приглушённый цвет)
- Рейтинг со звёздочкой и числом отзывов
- Сетка фото портфолио (3 колонки, квадратные), стрелка «ещё»
- Список услуг с ценами (без подробностей)
- `MainButton` Telegram с текстом «Записаться»

**Действия:**
- Нажать на фото портфолио → Экран «Портфолио» (увеличение + листание)
- Нажать на услугу → Экран 02 (выбор услуги уже выбран)
- «Записаться» (MainButton) → Экран 02

**Чего нет:** отзывы, описание мастера длинное, соцсети, кнопка «Написать»

## Уведомления (через бота, не Mini App)

| Событие | Кому | Текст |
|---------|------|-------|
| Новая запись | Клиент | «Вы записаны к Анне. Среда 4 июня в 14:00. Маникюр гель-лак.» |
| Новая запись | Мастер | «Новая запись: Мария Петрова, среда 14:00, гель-лак» |
| За 24 часа | Клиент | «Напоминаем: завтра в 14:00 маникюр у Анны. Отменить можно здесь 👇» |
| За 2 часа | Клиент | «Через 2 часа — ваш маникюр. Анна ждёт в 14:00 🕐» |
| Отмена клиентом | Мастер | «Мария Петрова отменила запись на среду 14:00» |
| Отмена мастером | Клиент | «Анна перенесла запись. Выберите другое время 👇» |

## Чего НЕ БУДЕТ в первой версии

- Оплата и предоплата (сложная интеграция — добавить в v2)
- Портфолио мастера с загрузкой фото (добавить в v2)
- Отзывы и рейтинг
- Чёрный список клиентов
- Аналитика (выручка за месяц, no-show статистика)
- Рассылки по базе клиентов
- Несколько мастеров / командная работа
- Интеграция с YClients / Google Calendar
- Программа лояльности и бонусы
- Прайс-лист с фото услуг

## Структура переходов (навигация)

```
Клиент:
  Профиль мастера (01)
    ├─ Нажал услугу ──────────→ Выбор услуги (02)
    └─ Нажал «Записаться» ────→ Выбор услуги (02)
  
  Выбор услуги (02)
    └─ Выбрал + MainButton ───→ Выбор даты/времени (03)
  
  Выбор даты/времени (03)
    └─ Выбрал + MainButton ───→ Подтверждение (04)
  
  Подтверждение (04)
    └─ «Записаться» ──────────→ Успех (04а)
  
  Успех (04а)
    ├─ «Мои записи» ──────────→ Мои записи (05)
    └─ «На главную» ──────────→ Профиль мастера (01)
  
  Мои записи (05)
    ├─ «Отменить» ────────────→ Подтверждение отмены
    └─ «Записаться снова» ────→ Выбор даты/времени (03)

Мастер:
  Дашборд (06)
    ├─ Запись в списке ───────→ Карточка клиента (09)
    ├─ «Расписание» ──────────→ Расписание (07)
    ├─ «Клиенты» ─────────────→ Список клиентов
    ├─ «Услуги» ──────────────→ Управление услугами (10)
    └─ «Настройки» ───────────→ Настройки расписания
  
  Расписание (07)
    ├─ Занятый слот ──────────→ Карточка клиента (09)
    ├─ Свободный слот ────────→ Добавить запись (08)
    └─ «+ Добавить» ──────────→ Добавить запись (08)
  
  Добавить запись (08)
    └─ «Создать» ─────────────→ Расписание (07)
  
  Карточка клиента (09)
    └─ «Записать снова» ──────→ Добавить запись (08)
```

## Технический стек

```
Фронтенд:   HTML + CSS + Vanilla JS
            Подключить: telegram-web-app.js (официальная библиотека Telegram)
            Никаких фреймворков в MVP — быстрее грузится

Бэкенд:     Node.js + Express
            База данных: SQLite (для старта, потом PostgreSQL)
            Бот: node-telegram-bot-api или grammy

Хранение:   localStorage для временных данных
            SQLite для записей, клиентов, услуг

Деплой:     Railway или Render (бесплатный тир для старта)
            Домен: обязателен (Telegram требует HTTPS для Mini App)
```

## Цветовая схема (Telegram-нативная)

Использовать переменные Telegram, не хардкодить цвета:

```css
/* Фон */
background: var(--tg-theme-bg-color);
/* Текст */
color: var(--tg-theme-text-color);
/* Вторичный текст */
color: var(--tg-theme-hint-color);
/* Ссылки и акценты */
color: var(--tg-theme-link-color);
/* Кнопки */
background: var(--tg-theme-button-color);
color: var(--tg-theme-button-text-color);
/* Карточки */
background: var(--tg-theme-secondary-bg-color);
```

Одна дополнительная переменная для beauty-акцента:
```css
--accent-beauty: #C9A96E; /* тёплое золото — работает и в тёмной, и в светлой теме */
```
