# TESTING.md — Руководство тестировщика

> Beauty Mini App · Telegram WebApp · версия MVP 1.0  
> Тестировщик может работать автономно: всё необходимое есть в этом файле.

---

## 1. Что это

Telegram Mini App для онлайн-записи к мастеру ногтевого сервиса.  
Два режима в одном приложении: **клиент** (записывается) и **мастер** (управляет расписанием).  
Бэкенда нет — все данные хранятся в `localStorage` браузера.

---

## 2. Как запустить

### Локально (браузер)
```
Открыть файл:  beauty-mini-app/tg-app/index.html
```
Никаких серверов, сборки и зависимостей не нужно. Открывается двойным кликом.

> ⚠️ Аватар мастера загружается с Unsplash — нужен интернет.  
> Фото портфолио (`images/nail1.jpg` … `nail6.jpg`) хранятся локально — работают offline.

### Задеплоенная версия (HTTPS)
```
https://nestas1985.github.io/vibe-coding/beauty-mini-app/tg-app/
```
Используй для теста с реального телефона и в Telegram.

## 5. localStorage — ключи

| Ключ | Что хранит | Как сбросить |
|------|-----------|--------------|
| `beauty_bookings` | Все записи (JSON-массив) | Удалить ключ |
| `onboarding_done` | Флаг «онбординг показан» | Удалить ключ |
| `offer_shown` | Флаг «оффер показан» | Удалить ключ |
| `notes_1`, `notes_2`, `notes_3` | Заметки о клиентах мастера | Удалить ключ |

**Как сбросить все данные:**  
DevTools (`F12`) → Application → Local Storage → выбрать все ключи → Delete

## 7. Тест-кейсы

### ТК-01: Онбординг — первый запуск

**Предусловие:** `localStorage` пуст (новая вкладка или сброшен)

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Открыть `index.html` | Экран онбординга (не главная) |

## 8. Проверка адаптивности

| Ширина | Что проверить |
|--------|--------------|
| 320px (iPhone SE) | Кнопки не обрезаются, текст не вылезает |
| 390px (iPhone 14) | Сетка портфолио 3/2 колонки выглядит ровно |
| 430px (максимум) | Приложение не растягивается шире |
| Горизонтальный режим | Не тестируется (приложение вертикальное) |

Кнопки: минимальная высота **52px** (основные), **44px** (второстепенные).  
Шрифт: минимум **12px** (подписи), основной текст **15px**.

## 9. Проверка тёмной темы

1. Открыть в Chrome DevTools → CSS → добавить на `<body>` класс `dark`
2. Все цвета должны переключиться: фон `#1c1c1e`, текст белый, карточки `#2c2c2e`
3. Акцент `#2AABEE` одинаков в обеих темах
4. В Telegram: тёмная тема передаётся автоматически через `themeParams`

## 10. Известные ограничения (не баги — MVP-решения)

| # | Ограничение | Где в коде |
|---|-------------|-----------|
| 1 | Все данные в localStorage — при очистке браузера исчезают | `loadBookings()` в app.js:393 |
| 2 | На тайм-лайне не отображаются слоты с `:30` (15:30 → не видно) | `buildTimeline()` в app.js:520 |
| 3 | Нажатие на запись в тайм-лайне всегда открывает клиента id=1 | app.js:535, `data-v="1"` хардкод |
| 4 | Нет реального бэкенда — бот не отвечает на /start, /help, /contact | Бот настроен через API, но webhook не установлен |
| 5 | Форма добавления записи (мастер) не подсвечивает пустые поля визуально | `submitManual()` в app.js:354 |
| 6 | `cancel-sheet` не закрывается если уйти на другой экран кнопкой «Назад» | `openCancelSheet()` в app.js:294 |
| 7 | Оплата и предоплата не реализованы | Запланировано в v2 |
| 8 | Напоминания (24ч / 2ч) не отправляются — нет бэкенда | Запланировано в v2 |
