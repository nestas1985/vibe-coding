# InstaTG — Критический разбор

> Записано для себя. К плану вернёмся позже.

---

## 1. Instagram-автоматизация — самое слабое место

- Новый аккаунт получает 50–200 просмотров на пост — алгоритм не доверяет новым
- Meta с 2024 года занижает охваты AI-генерированного контента
- В постах нет кликабельных ссылок — только "ссылка в bio". Stories со ссылками — только от 10 000 подписчиков
- Graph API не поддерживает Reels нормально, а Reels дают в 5–10 раз больше охвата чем карусели

---

## 2. Воронка Instagram → Telegram — конверсия будет ниже ожиданий

- Люди в Instagram не переходят по ссылкам — платформа для потребления, не для действий
- Реальная конверсия "увидел пост → запустил бота" — скорее 0.5–2%, не 3–8%
- Путь: увидел пост → bio → ссылка → Telegram → Start = 4–5 действий, на каждом теряется половина
- Аккаунт без лица и автора — ему не доверяют, сложнее монетизировать

---

## 3. Конкуренция — недооценена

- Fabulous, Headspace, BetterMe — уже в Instagram с аудиторией и бюджетами
- Telegram-ботов для трекинга привычек уже десятки
- Пока нет аудитории — воронка из Instagram не является преимуществом

---

## 4. Бизнес-модель — цифры не сходятся на старте

- Путь от 0 до 1000 активных пользователей органикой — 6–18 месяцев
- Telegram Stars берёт 30% комиссии: с 399₽ получаешь ~280₽
- При 1000 активных пользователях Claude API обходится $30–50+/мес только на Haiku
- Первые 20–50 платящих — не быстро при нулевой аудитории

---

## 5. Техническая сложность — это три проекта одновременно

- Instagram-автоматизация + генерация контента + Telegram-боты = три отдельных задачи
- APScheduler теряет все напоминания при рестарте сервера (нужна персистентность в БД)
- Надёжность бота критична — один сбой, и пользователь уходит навсегда

---

## 6. Правовые риски

- Instagram ToS запрещает автоматизацию — риск бана аккаунта в любой момент
- 152-ФЗ и GDPR: хранение привычек и расписания пользователей требует политики конфиденциальности

---
